# CP Review Skill

競技プログラミングの視点でC#コードをレビューする。

## レビュー観点

以下の順で分析し、日本語で回答する：

### 1. 計算量
- 時間計算量（Big-O記法）
- 空間計算量
- ボトルネックがあれば指摘

### 2. 正確性・エッジケース
- 空入力・最小ケース（長さ0, 1 など）
- 最大ケース（制約上限）
- 見落としがあれば具体的に指摘

### 3. コードの簡潔さ
- 無駄な処理・変数がないか
- よりシンプルに書ける部分があれば代替コードを提示
- C#のイディオムが活かされているか

### 4. 別解（あれば）
- より効率的・簡潔なアプローチがあれば概要を示す

## 出力フォーマット

```
### 計算量
- 時間: O(...)
- 空間: O(...)

### エッジケース
（問題あれば指摘、なければ「問題なし」）

### 改善提案
（改善できるなら差分を示す。なければ「なし」）

### 別解
（あれば概要を示す。なければ「なし」）
```
